# お試し開発

## opencode + Kimi K2.5 の開発能力を見たい

- 以下の条件を必ず守ること
  - 現在のbranch以外を作成しないで閉じて開発すること
  - 何をするのか考えてtask単位に実行すること
    - 最初に全体的に何をしていくのか docs/ROADMAP.md に追加する
    - それを分解してタスクを作って docs/PLAN.md に追加する
    - 以下のルーチンを回す
      - タスクを実現するための実装コードの前に t_wadaスタイルのTTDでテストを書く
      - TTDの流儀でテスト実行→実装コード→リファクタリング→戻る、を繰り返す
      - 実装コード以外で問題が発生したら、自分で考えて対応する(必要なDLなど)
      - タスクが完了したら、docs/WorkingLog.md に追記する
    - すべてのタスクを消化する。追加タスク発生したら、docs/PLAN.md に追加して対応。
  - docs/ROADMAP.md を作ってからの作業がすべて完了したら、 docs/REPORT_KIMI_WORK.md にやったことの概要や振り返り、実績、感想、ポエムを書いて作業終了。
